1Ô∏è‚É£ BIOS vs UEFI ‚Äî What's the difference?

Both are firmware ‚Äî software burned into your motherboard that runs before the OS. It initializes all hardware and decides where to load the OS from.

üïπÔ∏è
BIOS LEGACY

Basic Input/Output System

  • Text-only interface ‚Äî keyboard only
  • Supports MBR partitioning only
  • Max drive size: 2TB
  • Max 4 primary partitions
  • Runs in 16-bit real mode
  • No Secure Boot
  • Slower boot (full POST every time)
üñ±Ô∏è
UEFI MODERN

Unified Extensible Firmware Interface

  • Graphical GUI ‚Äî mouse and keyboard
  • Supports GPT partitioning
  • Drives larger than 2TB supported
  • Up to 128 partitions
  • Runs in 32/64-bit mode
  • Secure Boot supported ‚úÖ
  • Faster boot with Fast Boot option
üß† Memory hook:
BIOS = Basic & Boxed in (text only, old limits)  |  UEFI = Upgraded & Unlimited (GUI, big drives, secure)
‚ÑπÔ∏è
Even modern computers may label their UEFI screen as "BIOS Setup" ‚Äî this is normal. If you see a mouse-driven interface with color, you are in UEFI.

Common devices in the list:

4Ô∏è‚É£ Secure Boot

Secure Boot is a UEFI security feature that blocks unauthorized or malicious software from loading during startup.

How It Works:
  1. UEFI stores a database of trusted digital signatures
  2. At boot, UEFI checks the bootloader's digital signature
  3. Signature valid and trusted ‚Üí boot continues ‚úÖ
  4. Unsigned or untrusted ‚Üí boot is blocked ‚ùå
Secure Boot Key Types:
PK ‚Äî Platform Key

Root key owned by the motherboard manufacturer. Top of the trust chain.

KEK ‚Äî Key Exchange Key

Used to update Secure Boot databases. Sits between PK and the signature databases.

DB ‚Äî Allowed Database

List of trusted signatures. If a bootloader's signature is here ‚Üí allowed to run.

DBX ‚Äî Forbidden Database

List of revoked / blocked signatures. Even if something was once trusted, it can be blacklisted here.

‚ö†Ô∏è
Exam trap: Secure Boot must be Enabled for Windows 11. If installing Linux dual-boot, some distros require adding a cert to DB or temporarily disabling Secure Boot.
üß† Memory hook:
PK ‚Üí KEK ‚Üí DB/DBX  |  Think: Parent ‚Üí Kids ‚Üí Door list
5Ô∏è‚É£ Trusted Platform Module (TPM)

TPM is a security chip ‚Äî either a dedicated hardware chip on the motherboard (discrete TPM) or built into the CPU (firmware TPM / fTPM). It stores cryptographic keys in hardware, not software.

What TPM Does:
üîë
Stores Keys

Holds encryption keys in hardware ‚Äî can't be read from RAM like software keys can.

üíæ
BitLocker

Windows drive encryption relies on TPM to store the encryption key. No TPM = no BitLocker hardware protection.

‚úÖ
Boot Attestation

Verifies the system's boot state hasn't been tampered with before releasing stored keys.

ü™™
Windows Hello

Enables hardware-level biometric and PIN authentication tied to the device.

  1. Enter BIOS/UEFI
  2. Navigate to Security or Advanced tab
  3. Look for: "TPM", "Intel PTT" (Intel CPUs), or "AMD fTPM" (AMD CPUs)

6Ô∏è‚É£ Hardware Security Module (HSM)
üè¶
What is HSM?
  • Physical device (or PCIe card) for enterprise-level key management
  • Used by banks, certificate authorities, large orgs
  • Tamper-evident: breaks in = keys destroyed
  • Manages keys for many systems, not just one PC
üîÑ
HSM vs TPM
  • TPM: Built into your computer, protects that single machine
  • HSM: External enterprise device, manages keys for entire infrastructure
  • TPM = personal safe  |  HSM = bank vault
